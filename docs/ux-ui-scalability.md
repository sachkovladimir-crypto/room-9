# ROOM_9 UX/UI Scalability

ROOM_9 is currently a desktop-first web system. A full mobile redesign is not required for the current diploma scope. The priority is a clear, consistent desktop product with working role-based flows.

## Current UI Principles

- Dark brutalist aesthetic.
- Black background and off-white text.
- High contrast.
- Strong uppercase typography.
- Thin gray borders.
- Minimal cards.
- No gradients or decorative color systems.
- Clear hierarchy for discovery, profiles, dashboards, bookings, events, and streams.
- Reusable buttons, cards, forms, status pills, and empty states.

## Desktop UI Scalability

The UI can scale by turning repeated patterns into a consistent desktop design system:

- reusable DJ cards;
- reusable event cards;
- reusable stream cards;
- reusable form fields;
- reusable status pills;
- consistent page headers;
- consistent dashboard panels;
- consistent empty, loading, and error states;
- consistent filter blocks;
- reusable booking rows and chat panels.

Dashboard widgets can expand over time without redesigning the whole product. New roles can get their own dashboard sections while sharing the same navigation language, card system, status badges, and form controls.

## Future UX Additions

Future UX work should include:

- DJ onboarding;
- Organizer onboarding;
- Venue onboarding;
- Listener onboarding;
- advanced filters;
- saved searches;
- favorites;
- notification center;
- role-based dashboard navigation;
- admin moderation interface;
- desktop-first analytics views;
- multi-language support.

## Mobile Position

Mobile should not be the main build target right now. The current MVP should avoid catastrophic responsive breakage, but the diploma product is primarily judged as a desktop web system.

## UX Principle

ROOM_9 should scale by reusing a strict visual system: high-contrast typography, thin outlines, direct controls, and predictable role-based layouts.
